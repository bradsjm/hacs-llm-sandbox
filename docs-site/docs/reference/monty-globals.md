# Monty Globals

The exact prompt surface is generated by the integration, but the practical API centers on these globals and facades.

| Global or facade | Purpose |
| --- | --- |
| `hass` | Safe Home Assistant facade with `states`, `services`, `config`, `history`, `energy`, `logbook`, and `query`. |
| `states` | Convenience access to the safe state machine. |
| `er`, `dr`, `ar`, `fr`, `lr` | Registry-style facades for entity, device, area, floor, and label access. |
| `json`, `math`, `re` | Supported imports documented by the README. |

Awaitable facade calls are `await hass.history(...)`, `await hass.energy(...)`, `await hass.logbook(entity_ids=None, hours=None)`, `await hass.query(...)`, and gated `await hass.services.async_call(...)`. State, registry, config, and service-catalog reads are synchronous.

`hass.history` returns a flat list of `{entity_id, when, state, value}` rows (or flat analytics result dicts); it does not return the standalone `get_history` envelope or cursor. Raw results above 1000 rows are capped and reported in the top-level `overflow` field.
`hass.energy` accepts the same query arguments and returns the same production-shaped Energy data as `get_energy`. Its host-owned source is built from the fresh visibility-filtered snapshot; live Home Assistant objects and raw forecast configuration IDs never enter Monty.

`hass.logbook` returns copied JSON-safe activity entries for up to 20 visible entities over at most 24 hours. It returns at most the newest 200 entries in chronological order, has no cursor, and requires recorder and logbook runtime support.

Visible `State` records include Home Assistant state fields plus derived join keys `area_id`, `floor_id`, `device_id`, `platform`, and `unique_id`. Area, floor, label, and category records include `created_at` and `modified_at` when Home Assistant provides them. Config units include `temperature_unit`, `length_unit`, `mass_unit`, `pressure_unit`, `volume_unit`, `area_unit`, `wind_speed_unit`, and `accumulated_precipitation_unit`.

## Common code shapes

```python
result = [s.entity_id for s in hass.states.async_all() if s.domain == "light" and s.state == "on"]
```

```python
result = await hass.query(
    "select entity_id, state from states where domain = 'sensor' limit 20",
    hours=1,
)
```

```python
energy = await hass.energy(hours=168, period="day", include=["summary", "series"])
result = energy["summary"]["electricity"]["home_consumption"]
```

```python
entries = await hass.logbook("binary_sensor.front_door", hours=12)
result = [entry for entry in entries if "opened" in entry.get("message", "").lower()]
```
